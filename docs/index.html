    Scope of this dashboard: Each LT number is counted once, using only its first leg (the leg with the earliest actual departure time, i.e. Origin DC departure). Trip data is pulled on a schedule from the Linehaul Trip Compliance Tracker [SOC-LM] and [FM-SOC] "raw" tabs (combined, deduped where a trip appears in both). Only legs that had already actually departed and were billed with Cost Type "By Day" (day-rate rented vehicles) and departed from a genuine Origin DC (excludes First-Mile feeder legs starting at a collection Hub) are included. The most recent day is usually partial since it's still in progress at load time.

    Known gap: some vendors (notably Abimanyu) show markedly fewer trips under this source than the retired static CSV export did — the two trackers above may not cover every linehaul route category (e.g. Transit Point SOC–SOC shuttle runs), so a real vendor may be under-represented here if its routes fall outside SOC-LM/FM-SOC.

    Vendor Productivity — Trips per Onsite Daily Rent Unit

    DR = Daily Rent = Cost Type "By Day", applied consistently on both sides of this ratio. Daily Onsited = count of check-ins on the [29309] On Site Registration sheet with cost_type = "By Day" and arrival_status not "Expired" or "No Show" — i.e. vehicles that actually showed up, whether On Time, Late, or still Pending check-out. Deduped by distinct Nopol (plate number) + Vendor per day — if the same physical vehicle gets registered more than once on the same day, it counts once. Rows without a Nopol filled in count individually. Daily LT (using DR) = first-leg trips that had actually departed, billed Cost Type "By Day". Productivity = Daily LT ÷ Daily Onsited.

    Ordered DR vs Onsited DR, by Origin DC & Vehicle Type — per Day

    Each cell reads Ordered / Onsited. Ordered = units booked on the DR order sheet whose contract window (Start–End date) covers that day. Onsited = distinct vehicles (deduped by Nopol + Vendor) that checked in ("By Day", excl. Expired/No Show) at that DC on that specific day. The order sheet only has two coarse categories, CDD L and WB; WB is mapped to the onsite system's TRONTON (10WH) record, and CDD L covers every other Vehicle Type recorded onsite. The Campaign-to-date column sums Onsited across days, so it can exceed the campaign-to-date Ordered count.
