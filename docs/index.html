    Scope of this dashboard: Each LT number is counted once, using only its first leg (the leg with the earliest actual departure time, i.e. Origin DC departure).     Vendor Productivity — Trips per Onsite Daily Rent Unit

    DR = Daily Rent = Cost Type "By Day", applied consistently on both sides of this ratio. Daily Onsited = count of check-ins on the [29309] On Site Registration sheet with cost_type = "By Day" and arrival_status not "Expired" or "No Show" — i.e. vehicles that actually showed up, whether On Time, Late, or still Pending check-out. Note: this sheet has no plate number or registration ID, so unlike the earlier data source there is nothing to dedupe repeat check-ins by — each qualifying row counts as one onsite unit. If the same physical vehicle gets registered more than once the same day, it is counted more than once here. Daily LT (using DR) = first-leg trips that had actually departed, billed Cost Type "By Day". Productivity = Daily LT ÷ Daily Onsited.

    Ordered DR vs Onsited DR, by Origin DC & Vehicle Type — per Day

    Each cell reads Ordered / Onsited. Ordered = units booked on the DR order sheet whose contract window (Start–End date) covers that day. Onsited = check-ins ("By Day", excl. Expired/No Show) at that DC on that specific day — the current source sheet has no plate/ID to dedupe by, so a vehicle registered twice the same day counts twice. The order sheet only has two coarse categories, CDD L and WB; WB is mapped to the onsite system's TRONTON (10WH) record, and CDD L covers every other Vehicle Type recorded onsite. The Campaign-to-date column sums Onsited across days, so it can exceed the campaign-to-date Ordered count.
